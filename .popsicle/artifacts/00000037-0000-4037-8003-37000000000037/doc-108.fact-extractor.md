# PROJ-57 工作流目录 UI 事实基

## Summary

PROJ-57 为 cli-ux 桌面 UI 增加 **Workflows** 侧栏页与 `get_workflow_catalog` 只读 API，聚合 bundled/installed pipeline YAML 与 intent-coder `skill.yaml`，并与 Create Issue、Settings、PipelineView 联动。

## Bounded Contexts

| 上下文 | 路径 | 职责 |
|---|---|---|
| Workflow catalog（读模型） | `crates/cli-ux/src/workflow_catalog.rs` | 扫描 pipelines + skills，构建 `WorkflowCatalog` DTO |
| Tauri IPC | `crates/cli-ux/src/ui/commands.rs` | `get_workflow_catalog` command |
| UI 路由 | `ui/src/App.tsx`, `Sidebar.tsx` | `Page.kind === "workflows"` |
| Workflows 页 | `ui/src/pages/WorkflowsView.tsx` | Pipeline/Skills 双 Tab master-detail + ReactFlow DAG |
| 联动 | `IssuesView`, `SettingsView`, `PipelineView` | 预填 pipeline、画像链接、skill 跳转 |
| Pipeline 数据源 | `.popsicle/pipelines/*.yaml`, bundled in `crates/cli-ux/assets/pipelines/` | `PipelineDef` via `load_pipeline_def` |
| Skill 数据源 | `intent-coder/skills/*/skill.yaml` 或 `.popsicle/modules/intent-coder/skills/` | `skill.yaml` name/version/artifacts/workflow.states |

## Domain Glossary

- **Workflow catalog**：UI 只读目录，非 runtime 执行路径。
- **workflow_profile**：`project.yaml` 中画像，影响默认 pipeline 推荐 badge。
- **standalone skill**：不在 pipeline stage 中引用（如 `issue-author`）。

## Risk Hotspots

1. **双源 skills 目录**：workspace-root `intent-coder/` 覆盖 embedded module；catalog 与 runtime 须同源（`intent_coder_source()`）。
2. **侧栏第四项**：增加导航面，需 i18n + breadcrumb 同步。
3. **Create Issue 预填**：`Page` 扩展 `createPipeline`/`showCreate`；刷新后 state 可能丢失（可接受）。
4. **ReactFlow 静态 DAG**：catalog 页不可交互完成 stage（与 PipelineView 区分）。
