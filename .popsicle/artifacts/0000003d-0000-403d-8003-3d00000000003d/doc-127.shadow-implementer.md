# PROJ-62 工作区布局扁平化实现

## 范围

ADR-031 后 legacy 已清除；将 `.popsicle/self-host/` 内 `state.db` 与 `runs/` 上提至 `.popsicle/` 根，保留 `artifacts/`、`modules/`、`project.yaml` 不变。

## 实现摘要

| 路径 | 变更 |
|---|---|
| `crates/cli-ux/src/self_host.rs` | 扁平路径常量；打开时 `relocate_legacy_self_host_if_needed` + `cleanup_stale_legacy_self_host_subdir`；`admin relocate-workspace` |
| `crates/cli-ux/src/lib.rs` | AdminCommand + help |
| `crates/cli-ux/src/i18n.rs` | help 文案 |
| `crates/storage/src/sqlite.rs` | 注释对齐 |
| 测试 / golden / 活文档 | 路径 `.popsicle/state.db` |

## 目标布局

```text
.popsicle/
├── project.yaml
├── state.db
├── runs/
├── artifacts/
└── modules/intent-coder/
```

## 迁移语义

- 打开工作区：若存在 `.popsicle/self-host/state.db` 且无 flat `state.db` → 自动 rename 上提
- TSV：仍 import-only；legacy TSV 可在 self-host 子目录，导入后写入 flat
- `doctor`：`legacy_self_host_subdir` → `admin relocate-workspace`

## Divergence

无行为变更（Issue/Run/Doc 索引语义不变）；仅磁盘布局与路径字符串。
