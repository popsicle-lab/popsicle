# 产品辩论纪要 — telemetry

## Product Brief

**Telemetry** 是 Popsicle 的 **Agent 观测旁路**：OpenTelemetry WAL + OTLP 云端导出；与 issue/pipeline/doc **主平面解耦**（fail-open，不进 doc check，不用 IDE hook）。

## Participants 摘要

| 角色 | 结论 |
|------|------|
| PM | 价值：工作流调优、pipeline 耗时分析、Agent 质量回溯 |
| UXR | 维护者可选 completion score；不阻塞 stage complete |
| SEC | trace 默认 gitignore；redaction；OTLP TLS |
| ENGLD | 编排 auto-span + Agent 主动 tool run telemetry |
| OPS | 云端聚合；本地 WAL 可离线 |

## Decision

- 独立 product `telemetry` + crate `crates/telemetry/`
- 存储：`.popsicle/telemetry/{run_id}/`（不进 state.db）
- cli-ux：IO + `tool run telemetry`；skill-runtime：可选 pipeline span inject
- 三层：语义决策（artifact 正文）/ OTel span（旁路）/ 原始 JSONL WAL

## User Journeys（task 候选）

1. **admin**：配置 OTLP 端点、验证上云
2. **daily-ops**：Agent 上报 gen_ai span；编排 emit run/stage span
3. **troubleshooting**：诊断 export 失败（主流程不受影响）
