# 架构辩论纪要 — telemetry-otel-sidecar

## Topic

PDR-001 技术落地：OTel GenAI span、WAL 目录、OTLP 导出、cli-ux/skill-runtime inject 边界。[ADR 候选]

## Phase 1 — NFR

- 硬约束：fail-open；无 IDE hook；不进 doc check
- 优先级：可演进性 > 可靠性（主流程）> 导出延迟

## Phase 2 — 方案

- **A**（ARCH）：`crates/telemetry` + TelemetrySink trait；cli-ux 依赖 inject
- **B**：telemetry 逻辑全在 cli-ux — 否决：违反 ADR-007 shell 边界
- **C**：WAL 进 SQLite — 否决：ADR-013 职责分离

## Phase 3 — 评审

方案 A 全角色通过。

## Decision

采用 `crates/telemetry/`；OTLP HTTP/protobuf；WAL 为 OTLP JSON line envelope；TelemetrySink 同步 append + 异步 flush 线程（best-effort）。

## 关键分歧

- 同步 vs 异步 WAL：→ 同步 append（崩溃可恢复），flush 异步
