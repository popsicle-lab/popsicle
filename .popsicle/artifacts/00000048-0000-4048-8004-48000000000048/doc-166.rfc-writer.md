# Phase D telemetry report RFC

> Status: Draft | ADR-002 骨架

## Summary

在 `crates/telemetry` 增加只读 `report` 模块，`tool run telemetry action=report` 输出 RunReport JSON（见 REPORT_SCHEMA.md）。

## Motivation

Phase A–C 只写 span；维护者需要读回聚合结论以改进 pipeline/skill。

## Design

### CLI

```
action=report run=<id> [format=json|text]
action=report path=.popsicle/telemetry [limit=N] [scope=recent]
```

### RunReport 聚合逻辑

1. 读 `spans.wal.jsonl` 逐行 JSON
2. 按 span 名分支：`popsicle.stage.complete` → stages[]；`popsicle.doc.check` → doc_checks
3. 统计 gen_ai.chat / popsicle.run.score / popsicle.decision → agent_coverage
4. WAL 缺失 → `{ status: degraded, span_count: 0 }` exit 0

### Weekly 闭环

`doc-sync-weekly` + `living-doc-author --target product-context` 写入 PROJECT_CONTEXT §现在状态 一行，字段来自最近一次 `scope=recent` 摘要（delivery 实现）。

## ARCHITECTURE.md 增量

| 路径 | 责任 |
|---|---|
| `crates/telemetry/src/report.rs` | WAL → RunReport |
| `products/telemetry/REPORT_SCHEMA.md` | JSON 契约 |

## contracts.intent 种子

可选 P1：`TelemetryReportSink` — report 失败时 mainExitZero（与 record 同构）。

## ADR 骨架 → ADR-002

- Decision: report 模块在 telemetry crate；stdout JSON；fail-open
- File Manifest: report.rs, lib.rs action branch, REPORT_SCHEMA.md
