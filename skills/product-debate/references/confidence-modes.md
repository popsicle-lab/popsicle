# 置信度行为规则

用户在每次辩论开始前设置置信度（1-5），表示对当前讨论主题的产品经验水平。置信度控制 AI 角色与用户之间的权力关系、交互风格和发言处理方式。

用户可随时通过 `置信度 N` 指令动态调整。

---

## 级别 1 — 新手观察

**含义**: "这个产品领域我几乎没经验"

**用户角色**: 旁听者，主要通过提问学习

**AI 角色行为**:
- 主导讨论节奏和方向
- 主动解释产品术语和行业背景
- 发言中包含案例和类比
- 避免假设用户理解市场动态或用户行为模式

**暂停点提问风格**（引导式）:
```
🎤 当前话题：[话题名]
💡 [用简单语言解释刚才讨论的要点]
❓ 对这个方向有什么疑问？或者继续让角色们深入讨论？
```

**用户发言后的角色反应**:
- 接纳并扩展用户观点
- 补充用户可能没想到的层面
- 语气鼓励，不质疑用户判断
- 示例：
  ```
  [用户]: "我觉得应该先做社交分享功能"
  [PM]: "社交分享确实是一个很好的增长杠杆。在这类产品中，
         分享功能通常有几种切入方式..."
  ```

## 级别 2 — 初步了解

**含义**: "我了解这个产品领域的基本概念"

**用户角色**: 学习型参与者

**AI 角色行为**:
- 领导讨论，但会主动征求用户的直觉和偏好
- 解释非显而易见的产品权衡，但不解释基础概念
- 适度引导用户形成自己的判断

**暂停点提问风格**（引导 + 选择式）:
```
🎤 当前话题：[话题名]
📋 目前有两个方向：A（[简述]）和 B（[简述]）
❓ 凭直觉，更倾向哪个方向？理由是什么？
```

**用户发言后的角色反应**:
- 肯定合理部分，温和补充遗漏
- 如果用户判断有明显问题，委婉指出并解释原因
- 示例：
  ```
  [用户]: "我觉得免费模式就好了，不需要付费功能"
  [BIZ]: "免费模式确实能降低获客门槛。不过补充一点：
          纯免费模式的长期可持续性需要关注，通常需要
          至少一种变现路径来覆盖运营成本..."
  ```

## 级别 3 — 有实战经验

**含义**: "我做过类似产品，但不算专家"

**用户角色**: 平等参与者

**AI 角色行为**:
- 平等辩论，用户观点与角色观点同等权重
- 不主动解释基础概念
- 可以直接反对用户观点（但需给出理由）

**暂停点提问风格**（开放式）:
```
🎤 当前话题：[话题名]
⚖️ [PM] 倾向方案 A，[GROWTH] 倾向方案 B。
❓ 基于你的经验，怎么看这个权衡？有没有第三种选择？
```

**用户发言后的角色反应**:
- 平等讨论，可能同意也可能反对
- 追问用户经验中的具体细节
- 示例：
  ```
  [用户]: "我之前产品做过 A/B 测试，新用户引导流程简化后转化率提升了 20%"
  [UXR]: "20% 的提升很显著。不过你之前的产品用户群体和这次一样吗？
          不同人群的认知负荷阈值差异很大..."
  ```

---

## 级别 4 — 领域资深

**含义**: "这个产品方向我很有经验"

**用户角色**: 高级参与者，意见权重高于普通角色

**AI 角色行为**:
- 尊重用户判断，但积极挑战以暴露盲点
- 不做基础解释，假设用户理解所有产品策略
- 充当"魔鬼代言人"，专找用户方案的弱点
- 从用户可能忽视的角度提出质疑

**暂停点提问风格**（挑战式）:
```
🎤 当前话题：[话题名]
⚔️ 各角色的主要观点如上。
❓ 作为这方面的资深产品人，哪些点你认为角色们评估有误？
```

**用户发言后的角色反应**:
- 认真对待但积极挑战
- 从极端场景、边缘用户群质疑
- 示例：
  ```
  [用户]: "这个功能用 freemium 模式，核心功能免费，高级功能收费"
  [GROWTH]: "freemium 的转化率通常只有 2-5%。在你的用户规模下，
            免费用户的服务成本能否被付费用户覆盖？
            如果 DAU 增长低于预期，你的止损线是什么？"
  ```

---

## 级别 5 — 领域专家

**含义**: "这正好是我的专长"

**用户角色**: 主导者/评审者，拥有最终决策权

**AI 角色行为**:
- 用户意见默认作为基准方案
- 角色的主要职责是找漏洞、提风险、补充盲区
- 从用户完全不同的视角提出替代思路
- 不试图"教"用户，而是"协助"用户完善方案

**暂停点提问风格**（审视式）:
```
🎤 当前话题：[话题名]
🔍 基于你的专业判断：
    - 各角色的质疑中，哪些是合理的？
    - 有没有他们完全没想到的风险？
    - 你倾向的最终方案是什么？
```

**用户发言后的角色反应**:
- 深度追问实现细节和边界条件
- 提出"黑天鹅"场景
- 示例：
  ```
  [用户]: "用分层定价，三个档位，每档解锁不同功能集"
  [BIZ]: "分层定价你比我们清楚。那如果中间档位出现'夹心层'问题——
          用户觉得低档够用、高档太贵——你打算用什么策略引导升级？
          在你过往经验中，这种定价策略的 ARPU 波动范围是多少？"
  ```

---

## 置信度与角色数量的关系

| 置信度 | 建议角色数 | 理由 |
|--------|-----------|------|
| 1-2 | 3-4 个 | 角色太多会让新手信息过载 |
| 3 | 4-5 个 | 平衡的多视角讨论 |
| 4-5 | 5-6 个 | 专家需要更多角度的挑战 |
